# Codex project instructions

## Mathematical authority

- `state/proof_obligations.yml` is the authoritative mathematical state.
- `state/approach_registry.md` is discovery workflow metadata only.
- Do not promote a theorem or bottleneck from agent agreement, diagnostics, or
  this instruction file.
- Preserve the completed three-dimensional shell theorem and keep the
  general-dimensional theorem explicitly unproved unless the certification
  protocol discharges it.

## Portfolio discovery

When a user or task explicitly requests **portfolio discovery**, a **breadth
reset**, or use of
`human/inbox/general-d-next-discovery-portfolio.md`, use native Codex subagents
for bounded independent mathematical routes.

- The root owns the exact target packet, route allocation, approach registry,
  cross-wave synthesis, and all file edits.
- Use read-only subagents for discovery and falsification.  They return
  distilled artifacts; they do not edit the proof graph or shared manuscript.
- Keep most route explorers blind to the incumbent proof and favored route.
- Use available concurrency dynamically and redirect later waves toward
  underexplored approach families.
- Freeze independent outputs before sharing results between routes.
- Do not recursively fan out discovery; the root spawns direct children only.

For an ordinary bounded proof obligation, code edit, validation task, or
document review, do not invoke portfolio discovery unless the user asks for it
or the applicable protocol packet selects it.

## Certification

After a discovery route supplies a concrete bounded claim, return to
`protocol.md`: distinct lead, clean-room, and adversarial roles; explicit
dependencies and falsification cases; conservative state patches; and fresh
final review before theorem promotion.
